Video Games SQL Analysis

Final SQL Project | Video Game Sales Dataset

Project Overview

This SQL final project analyzes video game sales data in order to understand sales performance, regional preferences, platform trends, genre behavior and the relationship between scores and global sales.

The project focuses on writing business-oriented SQL queries that answer analytical questions relevant to publishers, developers and decision-makers in the gaming industry.

Business Questions

  • What factors most influence a game’s global sales success?
  • How do regional preferences differ across platforms and genres?
  • How many games were released on three or more platforms?
  • Which year had the highest number of genres reaching their sales peak?
  • Which platform recorded the highest year-over-year growth in global sales?

Tools Used

SQL Server • Excel • Data Analysis • Aggregations • Window Functions • Data Scaffolding • YoY Analysis

SQL Techniques Used

  • GROUP BY and aggregation functions.
  • COUNT(DISTINCT) to identify games released on multiple platforms.
  • RANK() to identify peak years by genre.
  • Weighted average, regular average and mode calculations.
  • Data scaffolding to create all Genre × Platform × Year combinations.
  • LAG() window function for year-over-year growth analysis.

Key Findings

  • 1,283 games were released on three or more platforms.
  • 2008 was the year with the highest number of genres reaching their peak global sales.
  • The ratings AO and K-A had identical weighted average, regular average and mode values because all games in those categories shared the same critic score.
  • The analysis shows that platform, genre, region and critic scores can all be used to evaluate video game sales performance.
  • Regional sales analysis helps identify where specific genres and platforms perform better.
